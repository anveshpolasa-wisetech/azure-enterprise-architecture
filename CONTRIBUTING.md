# Contributing to Azure Enterprise Architecture

Thank you for your interest in contributing to the Azure Enterprise Architecture project! This document outlines the guidelines and best practices to follow when contributing.

## Development Setup
- Clone the repository:
  ```
  git clone https://github.com/anveshpolasa-wisetech/azure-enterprise-architecture.git
  ```
- Install required dependencies:
  ```
  npm install
  ```

## Code Standards
- Follow the established coding standards for the project which include:
  - Consistent indentation (2 spaces)
  - Use of semicolons
  - Descriptive variable names

## Naming Conventions
- Use camelCase for variable and function names.
- Use PascalCase for class names.
- Name files according to their content.

## Documentation Requirements
- Ensure that all public functions and classes have appropriate documentation comments.
- Update any relevant documentation in the repository.

## Git Workflow
- Use feature branches for new development.
- Commit frequently to save your progress.

## Branching Strategy
- Use the following naming convention for branches:
  - `feature/description` for new features
  - `bugfix/description` for bug fixes

## Commit Message Format
- Use the following format for commit messages:
  - `type(scope): subject`  
    Example: `feat(auth): add login functionality`

## Pull Request Process
- Create a pull request against the `main` branch.
- Ensure your PR description clearly explains your changes.

## Code Review Checklist
- Ensure all tests pass before submitting your PR.
- Review the code for any potential improvements or issues.

## Unit and Integration Testing
- Write unit tests for new functionality.
- Ensure all existing tests pass before merging.

## Documentation Updates
- Update this `CONTRIBUTING.md` file as necessary to reflect any new processes.

## Release Process
- Follow SemVer (Semantic Versioning) for versioning the project.
- Update the Changelog with relevant changes for each release.

Thank you for contributing!
